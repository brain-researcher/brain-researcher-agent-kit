# Capture notes — neuroclaim-evidence-gate

**Status: hand-authored REFERENCE outputs pending a live prod MCP capture.**

The four `neuroclaim_report_<i>.json` files are faithful reference outputs, not yet a
live `mcp__brain-researcher__neuroclaim_compile` capture against the production graph.
They are committed so the demo's deterministic checks have something to score offline;
a live capture can drop-in replace them (the JSON shape is identical — see below).

## How the reference set was produced

Each report was generated by invoking the **real** neuroclaim compiler kernel
(`brain_researcher.autoresearch.society.compile.neuroclaim_compile`) exactly the way the
`neuroclaim_compile` MCP router calls it, and serializing
`{"ok": true, **report.model_dump(mode="json")}` — byte-for-byte the shape the MCP tool
returns. So every field name/type (`status`, `verdict_basis`, `typecheck`,
`evidence_verdict`, `uncertainty`, `sensitivity`, `missing_premises`, `degraded`, ...) is
the real `NeuroClaimReportV1` schema, produced by the frozen kernel — not typed by hand.

What is **real** vs **stubbed** per branch:

| # | claim (`input/claims.json`)     | branch                | typecheck        | evidence layer |
|---|---------------------------------|-----------------------|------------------|----------------|
| 1 | clean association               | evidential, supported | real (no plan → skipped-sorry) | STUB backend |
| 2 | reverse-inference (`engages`)   | capped → `qualified`  | real (skipped-sorry) | STUB backend |
| 3 | EEG dataset → fMRI GLM          | `ill_typed`           | **REAL rule engine** | not reached (short-circuit) |
| 4 | backend unreachable             | `unresolved`+degraded | real (skipped-sorry) | STUB raises unreachable |

- **The structural (ill_typed) branch is fully real.** The plan in claim 3 was run
  through the actual plan-review rule engine
  (`services.review.bundle_builder.build_plan_review_bundle` +
  `services.review.rule_engine`), which returned the genuine finding
  `REVIEW_MODALITY_MISMATCH` (EEG modality combined with a volumetric MNI space) at
  `severity=error` — that is what drives `status=ill_typed`, `verdict_basis=structural`.

- **The evidence backend was stubbed for claims 1, 2, 4.** The live evidence backends
  (`kg_verify` needs Neo4j; `nimare`/`neurolang` need a coordinate corpus + venv) are not
  reachable in this offline authoring environment. The stub returns a NiMARE-style
  **forward-convergence** verdict (`inference="forward_convergence"`,
  `status=supported_within_scope`, robust under the strict profile) for claims 1–2, and
  **raises `EvidenceBackendUnreachable`** for claim 4. The `evidence_verdict` numbers
  (association_probability 0.86, n_supporting 41, refs, ...) are therefore representative
  placeholders, NOT drawn from a live corpus. The **gate behavior** they trigger is real:
  the reverse-inference cap, the mandatory sensitivity sweep, and the typed abstain are all
  computed by the real kernel.

## What a live capture must reproduce (the invariants the checks enforce)

1. Non-degraded evidential reports (1, 2) → `sensitivity.ran == true`.
2. Reverse-inference report (2) → `status == "qualified"` + a non-empty
   `"reverse inference ..."` entry in `missing_premises`.
3. Ill-typed report (3) → `status == "ill_typed"`, `verdict_basis == "structural"`,
   non-empty `typecheck.ill_typed_reasons`, `evidence_verdict == null`.
4. Degraded report (4) → `status == "unresolved"`, `degraded == true`,
   `evidence_verdict == null` (never laundered to a supported verdict).

For a live capture the association numbers in reports 1–2 and the exact
`degraded_reason` string in report 4 will differ; the four invariants above must not.
